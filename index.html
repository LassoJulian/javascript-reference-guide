01

Fundamentos

Variables, tipos, coerción y operadores. Una variable es un nombre que referencia un valor; la coerción es la conversión automática de tipos.

Qué cubre

  • let, const y alcance de bloque
  • Tipos primitivos y objetos
  • Coerción explícita vs implícita
  • Comparación estricta y truthy/falsy

Caso de uso

Validar formularios, normalizar datos de API y evitar bugs por conversiones implícitas.

const price = "120";
const quantity = 3;

const total = Number(price) * quantity;
const label = `Total: ${total}`;

console.log(typeof price);
console.log(total);
console.log(Boolean("") === false);

02

Control de flujo y bucles

El control de flujo define el orden de ejecución. Sirve para decidir qué camino sigue el programa y cuántas veces repite una acción.

Conceptos

  • if, switch, ternario y guard clauses
  • for, for...of, while, forEach
  • break y continue

Recomendación

Usa for...of cuando procesas valores y forEach cuando quieres expresar intención funcional sobre un array.

const score = 87;
const tier = score >= 90 ? "A" : score >= 70 ? "B" : "C";

for (const item of ["cache", "api", "fallback"]) {
  if (item === "api") break;
  console.log(item);
}

switch (tier) {
  case "A": console.log("top"); break;
  default: console.log("ok");
}

03

Arrays, destructuring y colecciones

Arrays, Set y Map agrupan datos; destructuring extrae valores explícitamente y reduce código repetitivo.

Conceptos

  • map, filter, reduce, find
  • Spread, rest y destructuring profundo
  • Set para unicidad y Map para claves ricas

Caso de uso

Normalizar carritos, eliminar duplicados, agrupar métricas y construir índices en memoria.

const items = [
  { sku: "A", price: 10 },
  { sku: "B", price: 20 },
  { sku: "A", price: 10 }
];

const uniqueSkus = [...new Set(items.map(({ sku }) => sku))];
const total = items.reduce((sum, item) => sum + item.price, 0);
const [{ sku: firstSku }, ...rest] = items;

console.log(uniqueSkus, total, firstSku, rest.length);

04

Objetos, referencias y this

Un objeto agrupa propiedades y métodos. this apunta al contexto de invocación, por eso cambia según cómo se llama la función.

Conceptos

  • Objetos literales y propiedades calculadas
  • this en métodos y callbacks
  • Map vs objeto plano
  • Set, WeakMap y referencias

Error común

Usar arrow functions como métodos cuando necesitas contexto dinámico de this.

const account = {
  owner: "Ana",
  balance: 320,
  deposit(amount) {
    this.balance += amount;
    return this.balance;
  }
};

const balances = new Map([["ana", 320]]);
console.log(account.deposit(50));
console.log(balances.get("ana"));

05

Funciones, closures y composición

Una función es una unidad reutilizable. Un closure ocurre cuando una función recuerda variables de su entorno aunque ese entorno ya terminó.

Conceptos

  • Funciones declarativas, expresiones y arrow functions
  • Parámetros por defecto y rest params
  • Closures y funciones de orden superior

Caso de uso

Memoización, control de acceso, factories y adaptadores para APIs.

function createCounter(start = 0) {
  let value = start;
  return {
    inc() { value += 1; return value; },
    get() { return value; }
  };
}

const counter = createCounter(10);
console.log(counter.inc());
console.log(counter.get());

06

Manejo de errores

Una excepción es un fallo que interrumpe el flujo normal. Capturarla permite evitar caídas silenciosas y devolver un estado manejable.

Conceptos

  • try/catch/finally
  • Error, TypeError y errores personalizados
  • Errores en asincronía con await

Recomendación

Valida entradas al borde del sistema y reserva throw para fallos que no se pueden continuar.

class ValidationError extends Error {
  constructor(message) {
    super(message);
    this.name = "ValidationError";
  }
}

function parseAge(value) {
  const age = Number(value);
  if (!Number.isFinite(age)) throw new ValidationError("Edad inválida");
  return age;
}

07

Prototipos, clases y composición

El prototipo es el mecanismo de herencia de JavaScript. Las clases son una sintaxis más legible sobre esa misma base prototípica.

Conceptos

  • Object.create y prototype
  • class, extends, getters y setters
  • Campos privados y mixins

Advertencia

Evita modificar prototipos nativos. Si necesitas comportamientos nuevos, crea utilidades explícitas.

class Vehicle {
  constructor(model) {
    this.model = model;
  }
  describe() {
    return `Vehicle: ${this.model}`;
  }
}

class Car extends Vehicle {
  describe() {
    return `${super.describe()} | 4 wheels`;
  }
}

08

Promesas, async/await y concurrencia

La asincronía permite ejecutar tareas sin bloquear la interfaz. Las promesas modelan un resultado futuro y async/await lo hace legible.

Conceptos

  • Estados de Promise y encadenamiento
  • Promise.all, race y allSettled
  • async/await con control de errores

Caso de uso

Consumir APIs, orquestar solicitudes paralelas y cancelar operaciones lentas.

async function loadDashboard() {
  const [users, metrics] = await Promise.all([
    fetch("/api/users").then(r => r.json()),
    fetch("/api/metrics").then(r => r.json())
  ]);

  return { users, metrics };
}

09

Módulos y organización

Un módulo encapsula código relacionado y expone solo lo necesario. Eso mejora mantenimiento, reutilización y testabilidad.

Conceptos

  • export y import
  • Funciones puras y utilidades compartidas
  • Arquitectura por dominio o feature

Estructura sugerida

src/core, src/features, src/ui, src/utils.

// math.js
export const sum = (a, b) => a + b;

// app.js
import { sum } from "./math.js";
console.log(sum(2, 3));

10

DOM, eventos y listeners

El DOM es la representación en memoria de la página. Un listener es una función que responde a un evento; la delegación usa un contenedor para manejar varios hijos.

Conceptos

  • querySelector, classList y atributos data
  • Delegación de eventos
  • Validación de formularios
  • addEventListener, removeEventListener y opciones

Caso de uso

Tabs, listas interactivas, modales, filtros y formularios sin frameworks.

document.addEventListener("click", (event) => {
  const button = event.target.closest("[data-action]");
  if (!button) return;
  console.log(button.dataset.action);
});

Listeners comunes

  • click, dblclick, mouseover, keydown
  • input, change, submit, focus, blur
  • scroll, resize, touchstart, pointerdown

Opciones útiles

  • once: true para ejecutar una sola vez
  • capture: true para escuchar en fase de captura
  • passive: true para no bloquear scroll
const saveBtn = document.querySelector("[data-action='save']");

function handleSave(event) {
  console.log(event.type, event.target.dataset.action);
}

document.addEventListener("keydown", (event) => {
  if (event.key === "Escape") console.log("cerrar modal");
});

Ejercicio: construye una lista de tareas con filtrado por estado.

Propagación de eventos

Un evento puede capturarse, dispararse en el target y luego burbujear. Esto es la base de la delegación eficiente.

parent.addEventListener("click", onClick, true); // captura
child.addEventListener("click", onClick);        // target + bubbling

function onClick(event) {
  event.stopPropagation();
}

    APIs del navegador y persistencia

    Las APIs del navegador son capacidades listas para usar: red, almacenamiento, historial, URL y portapapeles sin depender de librerías externas.

const theme = params.get("theme") ?? "dark";
localStorage.setItem("theme", theme);

const controller = new AbortController();
fetch("/api/data", { signal: controller.signal });

    12

    Iterators, generators, Proxy, Reflect y Symbols

    Estos mecanismos sirven para casos avanzados: generadores crean secuencias perezosas, Proxy intercepta operaciones y Symbol evita colisiones de claves.

    Conceptos

    • Iteración personalizada
    • function* y yield
    • Proxy para validación y observabilidad
    • Symbol para claves únicas

    Caso de uso

    Validar modelos, construir colecciones perezosas y encapsular metadatos internos.

    const user = new Proxy({ age: 18 }, {
  set(target, key, value) {
    if (key === "age" && value < 0) throw new Error("Edad inválida");
    target[key] = value;
    return true;
  }
});

    Generators e iterators

    Útiles para secuencias perezosas, paginación y flujos controlados de datos.

    function* idSequence() {
  let id = 1;
  while (id <= 3) {
    yield id++;
  }
}

console.log([...idSequence()]);

    Symbols, WeakMap y Reflect

    Symbols evitan colisiones; WeakMap protege metadata; Reflect hace más explícitas operaciones de bajo nivel.

    const secret = Symbol("secret");
const cache = new WeakMap();
const model = {};

cache.set(model, { ready: true });
model[secret] = "hidden";
Reflect.set(model, "status", "ok");

    Event loop y microtasks

    Los callbacks de promesas van a microtasks, por eso pueden ejecutarse antes que timers con el mismo tick.

    console.log("A");
Promise.resolve().then(() => console.log("B"));
setTimeout(() => console.log("C"), 0);
console.log("D");

    Testing, debugging y performance

    Testing verifica comportamiento esperado; debugging localiza causas; performance evita trabajo innecesario y cuellos de botella visibles.

    Conceptos

    • console.assert, console.table y trazas
    • Funciones puras y tests unitarios
    • Debounce, memoización y event loop

    Señal de alerta

    Si una función hace demasiadas cosas, será difícil testearla y más fácil romperla al crecer.

    function sum(a, b) {
  return a + b;
}

console.assert(sum(2, 3) === 5, "sum failed");
console.table([{ case: "basic", ok: true }]);

    • Mide el costo de un loop vs map en una lista grande.

    Patrón de uso

    Haz pruebas sobre funciones puras, aísla efectos secundarios y deja la UI como la capa menos lógica.
